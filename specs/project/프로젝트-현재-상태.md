**백엔드 기술 스택**: Spring Boot

## 모델 상태

> **참고**: `ex_model.py`는 제외합니다.

모든 모델은 `src/models/common_model.py`의 `def_model` 함수를 사용하여 정의되며, 공통 필드(use_yn, shrn_yn, del_yn, crt_no, crt_dt, updt_no, updt_dt, del_no, del_dt)가 자동으로 추가됩니다.

#### 1. User (사용자)
- **파일**: `src/models/user_model.py`
- **테이블**: `users`
- **주요 필드**:

## 엔드포인트 기획

모든 엔드포인트는 RESTful API 설계 원칙을 따르며, 패스 파라미터는 `no` 형식을 사용합니다 (예: `:traitNo`, `:abilityNo`, `:charNo`, `:prjNo`).

### 공통 규칙

#### 응답 형식
- 모든 목록 조회: `SearchVo` 기반의 페이징 응답
- 상세 조회: 단일 객체 반환
- 생성/수정: 생성/수정된 객체 반환
- 삭제: 성공 여부 반환

#### 인증 및 권한
- **인증 불필요 (Public)**: 로그인, 회원가입, 비밀번호 재설정 요청
- **인증 필요 (Authenticated)**: 대부분의 조회, 생성, 수정, 삭제 엔드포인트
- **공유 여부 기반 접근 제어**: `shrn_yn = 'Y'`인 경우 모든 인증된 사용자 조회 가능, `shrn_yn = 'N'`인 경우 소유자 또는 어드민만 조회 가능
- **역할 기반 권한**: ADMIN은 모든 리소스 조회/수정 가능, USER는 자신이 생성한 리소스 또는 공유된 리소스만 조회 가능
